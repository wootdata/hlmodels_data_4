Fixing the Player Model Preview Color Remap
-------------------------------------------
Start with a size 160x200 (or 164x200) depending on where you look, @ 256 Index Color Mode (8bit).
Pal# 160-191 = Bottom Color (Color 2)
Pal# 192-223 = Top Color (Color 1)
Saving in GraphicsGale or Gimp will not allow SvenCoop to apply the color remap to the preview image even if the 256 color palette is saved correctly. 
I don't own Photoshop anymore so not sure of an alternative method for this, so we'll need to fix a line in a hex editor to make the remap magic happen for the preview image.
We can use web-based hex editor, https://hexed.it/ 
Drag your "model_name.bmp" file to the editor or just open in the file from the editor.

The 3rd line is where I make changes, denoted by: "00000020", it may look like the following:
"XX XX XX XX XX XX 00 00 00 00 00 00 00 00 00 00" where X = any hexadecimal from 0-F. 

We need to change some of the 00 values such that it looks like the following:
"XX XX XX XX XX XX 13 0B 00 00 13 0B 00 00 00 01" leave whatever was in the X values as they were.
Now export the BMP file which will just download it. Make a back up of the original and try adding the newly modified one in. See if the preview works and remaps in SvenCoop.

I have no idea why this happens, what the hexadecimal changes mean, or why changing that section to "13 0B 00 00 13 0B 00 00 00 01" fixes the pal remap. However I've been successful every time with this method to modify my saved bmp images from GraphicsGale.
